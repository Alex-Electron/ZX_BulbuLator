# Шаг 0 — Настройка: от «голой» платы до прошитого битстрима

Languages: [English](README.md) · **Русский**

Это руководство для тех, кто начинает с нуля. К концу у тебя будет плата под напряжением, подключенный к ней JTAG-кабель, установленное ПО и прошитый готовый битстрим из Шага 1 — два мигающих светодиода. Опыт работы с FPGA не требуется.

Сразу честно предупредим: если ты купил EBAZ4205 в виде «голой» платы, придётся немного попаять (резистор режима загрузки, а иногда и разъём JTAG, и гнездо MicroSD). Если у тебя всё это уже есть, пропусти эти шаги.

## Что тебе понадобится

- Плата **EBAZ4205**, вариант Zynq-7010 (`XC7Z010`).
- **Источник питания** — 5 В или 12 В (см. ниже).
- **JTAG-программатор**, один из следующих:
  - кабель, поддерживаемый Vivado (Xilinx Platform Cable, Digilent HS2/HS3) — самый простой вариант, или
  - **Raspberry Pi Pico** плюс 5 соединительных проводов — дешевый вариант, который используется в этом проекте.
- **Карта MicroSD** (любого объёма) с загрузочным образом (см. шаг 2).
- **Паяльник** с тонким жалом для подготовки платы, как описано ниже.
- Компьютер. Linux лучше всего подходит для работы с демоном Pico; Windows подойдет для
  использования графического интерфейса Vivado.

## 1. Подключение питания к плате

EBAZ4205 — это бывшее майнинговое оборудование, поэтому с питанием придётся немного повозиться. Два распространённых варианта:

- **5 В** на контакты питания разъёма JTAG/UART (потребуется небольшой диод Шоттки
  или перемычка), либо
- **12 В** на разъём вентилятора/питания — **следи за полярностью**, обратное подключение
  может вывести плату из строя.

Когда питание подано, на плате загорается зелёный светодиод. Не продолжай, пока он не загорится.

## 2. Режим загрузки

EBAZ4205 выбирает источник загрузки по двум контактам — IO0 (MIO5) и IO2 (MIO4) — которые настраиваются с помощью нескольких резисторов рядом с микросхемой NAND:

| Режим | IO0 (MIO5) | IO2 (MIO4) |
|------|:----------:|:----------:|
| JTAG | 0 | 0 |
| NAND | 0 | 1 |
| SD   | 1 | 1 |

![Схема бут-страпа EBAZ4205 и таблица истинности](images/Boot-Mode.jpg)

По умолчанию плата находится в режиме **NAND**: резистор `R2584` (20 кОм) подтягивает IO0 к GND, а резистор `R2578` (20 кОм) — IO2 к Vcc. Чтобы перейти в режим **SD**, нужно только, чтобы IO0 был в высоком состоянии (IO2 и так уже в высоком), поэтому добавь подтяжку для IO0 на **R2577** — пустое гнездо «NC» прямо рядом с R2584.

В этой статье резистор переносят с R2584 на R2577. Не обязательно так аккуратно: **я просто соединил R2577 капелькой припоя, и устройство загрузилось с SD с первой попытки** — надёжный подтягивающий резистор к Vcc лучше, чем 20k, опускающий на GND.

![Расположение R2577 и R2584 на плате, увеличенное](images/r2577-location.jpg)

*R2577 (подтягивающий резистор для SD, который нужно добавить/замостить) и заводской R2584 под лупой.*

Две вещи, которые упрощают жизнь:

- **Если MicroSD не вставлена, плата автоматически переходит в режим JTAG.**
  Для работы исключительно через JTAG можно не вставлять карту и вообще не трогать резисторы.
- Разъём MicroSD **на большинстве плат уже установлен** — паять там не нужно.

Для мигания в шаге 1 битстрим поступает через JTAG, так что загрузочная карта пока не нужна; создание загрузочного образа на SD — это следующий шаг.

В следующий раз я бы предпочёл переключать режимы загрузки с помощью **переключателя или перемычки**, а не перепаивать — автор статьи подключил два SPDT-переключателя именно для этого. Паяный мостик на R2577 пока что постоянный, но это нормально.

Подробности о загрузочном цикле и схема из [руководства по EBAZ4205 на сайте theokelo.co.ke](https://theokelo.co.ke/getting-starting-with-ebaz4205-zynq-7000/).

## 3. Установи программное обеспечение

Мы собираем и прошиваем с помощью инструментов AMD/Xilinx версии **2023.1**. Более новые версии тоже должны работать — здесь нет ничего, что зависело бы от конкретной версии. Всё это бесплатно при наличии учётной записи AMD/Xilinx.

Для чего нужен каждый компонент:

- **Vivado Lab Edition** — отдельный Hardware Manager. Этого достаточно,
  чтобы **записать** готовый битстрим через JTAG. Небольшой файл.
- **Vivado ML Standard** — полная версия инструмента для **компиляции** битстрима из исходного кода
  (каждый шаг поставляется с файлом `build.sh`). Большой размер (десятки ГБ).
- **Vitis** — «голый» ARM. Начиная с шага 7, Cortex-A9 выполняет реальную работу (
  управляющая плоскость AXI, загрузчик с SD-карты, OSD), и любому образу для загрузки с SD-карты нужен
  FSBL. Vitis предоставляет `xsct` и компилятор `arm-none-eabi`, которые их создают.
- Для образов SD-boot (`BOOT.BIN`) **никаких дополнительных инструментов** не требуется — скрипт `flash/build_boot.sh` на каждом этапе создаёт их с помощью `bootgen`, который поставляется вместе с Vivado. (Он передаёт `bootgen` предварительно извлечённые `.bin`-разделы и исправляет заголовок BootROM, что позволяет обойти сегфолт `bootgen`/glibc-2.43 на самых новых хостах.)

Ещё одно: при первой сборке из исходников — загрузи общий HDL-файл, на котором основаны эти шаги, из корня репозитория:

```
./get_deps.sh
```

Он клонирует наши два ядра (ядро Atlas ZX и ядро hdl-util HDMI) и IP-блок Digilent `rgb2dvi`, причём каждое из них **привязано к конкретному коммиту**, в папки `cores/` и `deps/`, которые игнорируются git. Запускать его нужно только один раз. (ПЗУ Spectrum загружается отдельно, на каждом этапе, скриптом `get_rom.sh` — `assemble.sh` запускает его за тебя.)

## 4. Подключи JTAG-программатор

### Вариант A — кабель, поддерживаемый Vivado (самый простой)

Подключи JTAG-кабель Digilent или Xilinx к разъёму JTAG на плате. В Hardware Manager в Vivado воспользуйся функцией **Auto Connect**. Перейди к шагу 5.

### Вариант B — Raspberry Pi Pico (дешёво, мы используем именно его)

**Прошивай прошивку Pico:** удерживай кнопку BOOTSEL на Pico, подключи его к USB — он отобразится как диск `RPI-RP2`. Скопируй файл [`firmware/xvcPico_v2_soft_edges.uf2`](firmware/) на этот диск. Pico перезагрузится в режиме JTAG-адаптера. (Это сборка «soft edges» — с медленной скоростью нарастания сигнала и током 2 мА — именно благодаря ей плотные битстримы записываются без ошибок BAD_PACKET. Источник: ветка `zynq-dense-bitstreams` форка, также доступная в основном репозитории как [kholia/xvc-pico#3](https://github.com/kholia/xvc-pico/pull/3).)

**Подключи Pico к разъёму JTAG на плате.** Разъём JTAG — это **J8** (контакты с надписями TDI, TDO, TCK, TMS, VCC). Если тебе понадобится последовательная консоль позже, она подключается к разъёму **J7** (VCC, RXD, TXD, GND).

![Разъёмы EBAZ4205 J7 (UART) и J8 (JTAG)](images/uart-jtag.jpg)

И Pico, и плата работают при напряжении 3,3 В, поэтому их можно подключать напрямую, а названия сигналов совпадают — перекрещивания TDI/TDO нет.

| Сигнал JTAG | Pico   | EBAZ4205 J8 |
|-------------|--------|-------------|
| TDI | GPIO16 | TDI |
| TDO | GPIO17 | TDO |
| TCK | GPIO18 | TCK |
| TMS | GPIO19 | TMS |
| GND | вывод 23 | GND |

Не нужно подключать VCC разъёма J8 к Pico — достаточно четырёх сигналов и общей массы. (Фото в заголовке взято из примечаний к EBAZ4205 в проекте [xvc-pico](https://github.com/Alex-Electron/xvc-pico).)

**Запусти демон XVC на хосте.** Он соединяет USB-порт Pico с виртуальным кабелем Xilinx через TCP-порт 2542. Скомпилируй его из репозитория [xvc-pico](https://github.com/Alex-Electron/xvc-pico):

```
sudo apt install cmake gcc-arm-none-eabi libnewlib-arm-none-eabi \
  libstdc++-arm-none-eabi-newlib git libusb-1.0-0-dev build-essential make g++ gcc
git clone https://github.com/Alex-Electron/xvc-pico.git
cd xvc-pico/daemon && cmake . && make
./xvcd-pico        # turn the Pico on first
```

Чтобы запустить программу **без sudo**, один раз установи входящее в комплект правило udev и переподключи Pico:

```
sudo cp 99-programming-adapters.rules /etc/udev/rules.d/
sudo udevadm control --reload && sudo udevadm trigger
```

## 5. Запроши битстрим

Битстрим для мигания уже готов: [`../01-board-bringup-blink/blink_z010.bit`](../01-board-bringup-blink/).

### Самый простой способ — официальный Vivado Hardware Manager (GUI)

1. Открой Vivado (или Vivado Lab) → **Hardware Manager** → **Open Target**.
2. Выбери кабель:
   - Вариант A (реальный кабель): **Auto Connect**.
   - Вариант B (Pico): **Add Xilinx Virtual Cable (XVC)**, хост `localhost`, порт `2542`.
3. Появится `xc7z010`. Щелкни по нему правой кнопкой мыши → **Program Device** → выбери
   `blink_z010.bit` → **Program**.
4. Дождись появления сообщения `End of startup status: HIGH`.

### С помощью скрипта (Linux + Pico) — необязательно

```
bash ../01-board-bringup-blink/jtag_flash.sh ../01-board-bringup-blink/blink_z010.bit
```

Скрипт (пере)запускает стек `xvcd-pico` + `hw_server`, открывает целевую платформу XVC, находит микросхему `xc7z010` и записывает в неё программу. Если у тебя пути отличаются, переопредели их с помощью переменных среды `XVCD_PICO` и `VIVADO_LAB`.

## 6. (Необязательно) Скомпилируй битстрим самостоятельно

Если полная версия Vivado добавлена в `PATH`, каждый этап сборки выполняется одной командой — `./build.sh`. Самые простые этапы не имеют внешних зависимостей:

```
cd ../01-board-bringup-blink
./build.sh                 # -> blink_z010.bit
```

Для этапов HDMI (3, 4) и ZX (6–9) сначала нужно один раз загрузить общие ядра (раздел 3):

```
./get_deps.sh              # from the repo root, one time
cd research/06-zx-spectrum-128
./build.sh                 # assembles sources/build/ from the cores + this step's delta, then builds
```

Для этапов ZX скрипт `build.sh` запускает `assemble.sh`, который собирает исходники данного этапа в папку `sources/build/` (каждый этап поставляет только то, что он изменяет, а остальное берет из предыдущих этапов — в его README точно указано, что именно). Целевой компонент — `xc7z010clg400-1`. Полный битстрим для 7010 занимает около 2 083 856 байт; этапы для DDR (8, 9) меньше — примерно 1,09 МБ. Этот размер — удобный способ проверить, что сборка прошла для 7010.

## Результат

Два светодиода (D18 и H18) мигают поочерёдно с частотой примерно 1–2 Гц. Вот и весь инструментарий, работающий от начала до конца. Перейди к [Шагу 1](../01-board-bringup-blink/), чтобы узнать, что на самом деле делает эта конструкция и почему она построена именно так.

## Справочная информация по плате расширения

Плата расширения (адаптер EBAZ4205 с AliExpress) оснащена разъёмом HDMI, четырьмя светодиодами, четырьмя кнопками и разъёмом 5 В постоянного тока и подключается к краевым разъёмам DATA1/DATA2/DATA3 основной платы.

![Плата расширения EBAZ4205: HDMI, разъём 5 В, кнопки, светодиоды, промаркированные GPIO](images/expansion-board.jpg)

**HDMI имеет схему подключения «семейства B» — тактовый сигнал TMDS на F19/F20.** Для этих адаптеров существует две несовместимые распиновки HDMI; другая («семейство A», тактовый сигнал на H16/H17) в данном случае приводит к появлению пустого экрана, так как на этой плате H16/H17 заняты UART. В шаге 3 используется семейство B.

| Функция | Контакты FPGA |
|----------|-----------|
| Тактовый сигнал TMDS | F19 / F20 |
| Данные TMDS 0/1/2 | D19/D20, C20/B20, B19/A20 |
| LED1..LED4 | D18, H18, E19, K17 (активный уровень высокий) |
| KEY1..KEY4 | P19, T19, U20, U19 (активный уровень низкий) |

Всё проверено на аппаратном обеспечении. Битстримы сопоставляют **контакты FPGA** с функциями, а не с надписями на печатной плате.
