# Шаг 4 — HDMI с рисунками, переключаемыми кнопками

Languages: [English](README.md) · **Русский**

На шаге 3 на экране появилось неподвижное изображение. Теперь добавим интерактивность: четыре кнопки на контроллере Shield меняют то, что рисуется. Тот же путь видеосигнала, что и на шаге 3 (PS7 → FCLK0 → MMCM → rgb2dvi), тот же рецепт для Flash — только генератор узоров стал более разнообразным.

## Что на экране

Тёмно-синее **окно размером 960×720, центрированное на экране**, с прыгающим квадратом 144×144 (режим по умолчанию), цвет которого меняется с каждым кадром. Четыре кнопки:

| Кнопка | Вывод | Нажми, чтобы… |
|--------|-----|-----------|
| KEY1   | P19 | переключать рисунок: квадрат → цветные полосы → градиент → шахматная доска |
| KEY2   | T19 | инвертировать цвета |
| KEY3   | U20 | замедлить анимацию |
| KEY4   | U19 | ускорить анимацию |

Кнопки срабатывают по фронту сигнала с временем подавления дребезга ~14 мс, поэтому однократное нажатие действует один раз (а не по принципу «удерживай, чтобы сработало», как в шаге 2). Сопоставление осуществляется по выводам FPGA; физические надписи на кнопках на расширении могут отличаться (см. [Шаг 0](../00-setup/#expansion-board-reference)).

## Почему окно центрировано, а не занимает весь экран

*Сигнал* HDMI имеет полное разрешение 1280×720, но в демо-версии контент отображается только в центрированной области размером **960×720** (именно **4:3**), а по бокам остаётся чёрный фон — так называемый «pillarboxing». Ретро-контент имеет формат 4:3; если растянуть его на панель 16:9, всё будет выглядеть растянутым, поэтому чёрные боковые полосы сохраняют правильные пропорции, как на ЭЛТ-мониторе. Чтобы вместо этого использовать полную ширину, установи `WX0=0` и `WW=1280` в [`pattern_gen3.v`](pattern_gen3.v) и перекомпилируй.

## Сборка

Такая же схема блоков, что и в шаге 3 (здесь повторно используется `clkgen2.v`, MMCM `clk_wiz` и `rgb2dvi` от Digilent), с [`pattern_gen3.v`](pattern_gen3.v) в качестве источника шаблона и [`hdmi_btn720.xdc`](hdmi_btn720.xdc), добавляющим четыре вывода кнопок.

Сначала загрузи все зависимости из корня репозитория, а потом запусти сборку:

```
../../get_deps.sh        # fetches rgb2dvi into deps/vivado-library, pinned (once for the whole repo)
./build.sh               # → hdmi720_z010.bit
```

Компонент `xc7z010clg400-1`, размер выходных данных — ~2 083 867 байт. В комплект входит готовый файл `hdmi720_z010.bit`.

## Прошивка

Тактовая частота PS такая же, как в шаге 3, поэтому используется тот же надёжный двухэтапный алгоритм (запрограммируй PL с помощью `vivado_lab`, а затем запусти `ps7_init` + `ps7_post_config` через `xsdb`):

```
bash flash_demo.sh hdmi720_z010.bit
```

`Статус завершения запуска: HIGH`, затем `PS7_INIT_DONE` и `PS7_POST_CONFIG_DONE`. Почему именно такая последовательность — смотри в [Шаге 3](../03-hdmi-bars/), а о настройке JTAG — в [Шаге 0](../00-setup/). При обычной загрузке с SD-карты FSBL сам поднимает тактовую частоту PS.

## Ожидаемый результат

В центре окна появляется прыгающий квадрат, а индикатор H18 мигает. Кнопка KEY1 переключает четыре шаблона, KEY2 — инвертирует изображение, а KEY3/KEY4 — меняют скорость. Ввод данных плюс реальное изображение — теперь плата действительно интерактивна.
